01
Uma classe declarada com abstract pode conter métodos concretos (com implementação) além de métodos abstratos.
02
Um método abstract DEVE ter corpo (implementação) dentro da própria classe abstrata.
03
Uma subclasse concreta que estende uma classe abstrata DEVE implementar todos os métodos abstratos herdados.
04
Em Java, uma classe abstrata pode ter construtor.
05
Uma variável do tipo de uma classe abstrata pode referenciar objetos de suas subclasses concretas (polimorfismo).
06
Se uma subclasse não implementar todos os métodos abstratos da superclasse, ela deve ser declarada também como abstract.
07
Uma classe pode conter métodos abstract sem ser declarada como abstract class.
08
Uma classe abstrata pode estender (extends) outra classe abstrata.
09
Uma classe abstrata pode ter 0 (zero) métodos abstratos — a palavra abstract na classe apenas impede sua instanciação direta.
10
A palavra-chave abstract pode ser usada tanto para classes quanto para métodos em Java.
