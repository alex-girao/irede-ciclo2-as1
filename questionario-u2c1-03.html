Criar uma Classe com Atributos
Básico
Crie uma classe chamada Produto com três atributos públicos. Em seguida, no main, instancie um objeto, atribua valores e imprima cada campo.
Requisitos
  • Declare os atributos String nome, double preco e int quantidade
  • Instancie com Produto p = new Produto()
  • Atribua: nome "Caderno", preco 12.90, quantidade 5
  • Imprima cada atributo em uma linha separada
Solução
public class Produto { String nome; double preco; int quantidade; } public class Ex01CriarClasse { public static void main(String[] args) { Produto p = new Produto(); p.nome = "Caderno"; p.preco = 12.90; p.quantidade = 5; System.out.println("Produto: " + p.nome); System.out.println("Preço: R$ " + p.preco); System.out.println("Qtd: " + p.quantidade); } }
Saída esperada no console
Produto: Caderno Preço: R$ 12.9 Qtd: 5
2)
Adicionar Método à Classe
Básico
Adicione à classe Produto um método exibir() que imprime os dados formatados. Crie um objeto e chame o método.
Requisitos
  • Crie o método void exibir() dentro da classe Produto
  • O método deve imprimir uma linha === Produto === seguida de nome, preço e quantidade
  • Instancie um objeto, atribua valores e chame p.exibir()
Solução
public class Produto { String nome; double preco; int quantidade; void exibir() { System.out.println("=== Produto ==="); System.out.println("Nome: " + nome); System.out.println("Preço: R$ " + preco); System.out.println("Quantidade: " + quantidade); } } public class Ex02MetodoExibir { public static void main(String[] args) { Produto p = new Produto(); p.nome = "Mochila"; p.preco = 89.90; p.quantidade = 3; p.exibir(); } }
Saída esperada no console
=== Produto === Nome: Mochila Preço: R$ 89.9 Quantidade: 3
3)
Atributo Privado e Getter
Básico
Crie a classe ContaBancaria aplicando encapsulamento: declare o atributo saldo como privado e forneça um getter público para leitura.
Requisitos
  • Declare private double saldo na classe ContaBancaria
  • Crie o getter public double getSaldo() que retorna o saldo
  • Instancie um objeto e imprima o saldo inicial (Java inicializa double como 0.0)
Solução
public class ContaBancaria { private double saldo; // atributo encapsulado public double getSaldo() { return saldo; } } public class Ex03AtributoPrivado { public static void main(String[] args) { ContaBancaria conta = new ContaBancaria(); System.out.println("Saldo inicial: R$ " + conta.getSaldo()); } }
Saída esperada no console
Saldo inicial: R$ 0.0
4)
Setter com Validação
Básico
Adicione à classe ContaBancaria um setter que valida o valor antes de atribuir: somente aceita saldo maior ou igual a zero; caso contrário, exibe uma mensagem de erro.
Requisitos
  • Crie public void setSaldo(double saldo)
  • Use if (saldo >= 0) para validar; dentro, use this.saldo = saldo
  • No else, imprima "Erro: saldo não pode ser negativo!"
  • No main, chame setSaldo(1500.0) e depois setSaldo(-100.0)
Solução
public class ContaBancaria { private double saldo; public double getSaldo() { return saldo; } public void setSaldo(double saldo) { if (saldo >= 0) { this.saldo = saldo; } else { System.out.println("Erro: saldo não pode ser negativo!"); } } } public class Ex04SetterValidado { public static void main(String[] args) { ContaBancaria conta = new ContaBancaria(); conta.setSaldo(1500.0); System.out.println("Saldo: R$ " + conta.getSaldo()); conta.setSaldo(-100.0); } }
Saída esperada no console
Saldo: R$ 1500.0 Erro: saldo não pode ser negativo!
Dica: Use this.saldo dentro do setter para referenciar o atributo da classe e diferenciar do parâmetro de mesmo nome. Sem o this, Java entenderia que você está atribuindo o parâmetro a ele mesmo.
5)
Método que Reutiliza o Setter
Básico
Adicione à ContaBancaria o método depositar() que reutiliza internamente o setter para atualizar o saldo com segurança.
Requisitos
  • Crie public void depositar(double valor)
  • Se valor > 0, chame setSaldo(saldo + valor) e imprima a confirmação
  • Caso contrário, imprima "Valor inválido!"
  • No main: defina saldo 500.0, deposite 300.0, tente depositar -50.0 e imprima o saldo final
Solução
public void depositar(double valor) { if (valor > 0) { setSaldo(saldo + valor); // reutiliza a validação do setter System.out.println("Depósito: R$ " + valor); } else { System.out.println("Valor inválido!"); } } // No main: ContaBancaria conta = new ContaBancaria(); conta.setSaldo(500.0); conta.depositar(300.0); conta.depositar(-50.0); System.out.println("Saldo final: R$ " + conta.getSaldo());
Saída esperada no console
Depósito: R$ 300.0 Valor inválido! Saldo final: R$ 800.0
6)
Construtor com Parâmetros
Básico
Adicione à classe Produto um construtor que recebe os três atributos como parâmetros. Use this para atribuir os valores. Crie um objeto usando o construtor e chame exibir().
Requisitos
  • Crie public Produto(String nome, double preco, int quantidade)
  • Dentro do construtor, use this.nome = nome, this.preco = preco, this.quantidade = quantidade
  • No main, instancie com new Produto("Caneta", 3.50, 10) e chame exibir()
Solução
public class Produto { String nome; double preco; int quantidade; public Produto(String nome, double preco, int quantidade) { this.nome = nome; this.preco = preco; this.quantidade = quantidade; } void exibir() { System.out.println("=== Produto ==="); System.out.println("Nome: " + nome); System.out.println("Preço: R$ " + preco); System.out.println("Quantidade: " + quantidade); } } public class Ex06Construtor { public static void main(String[] args) { Produto p = new Produto("Caneta", 3.50, 10); p.exibir(); } }
Saída esperada no console
=== Produto === Nome: Caneta Preço: R$ 3.5 Quantidade: 10
Regras do construtor: O nome deve ser idêntico ao da classe; não possui tipo de retorno (nem void); pode receber parâmetros para definir valores iniciais.
7)
Múltiplos Objetos da Mesma Classe
Básico
Usando o construtor criado no exercício anterior, instancie dois objetos Produto diferentes e chame exibir() em cada um, demonstrando que uma mesma classe gera objetos independentes.
Requisitos
  • Crie p1: new Produto("Caderno", 12.90, 5)
  • Crie p2: new Produto("Caneta", 3.50, 10)
  • Chame p1.exibir() e p2.exibir()
Solução
public class Ex07MultiplosProdutos { public static void main(String[] args) { Produto p1 = new Produto("Caderno", 12.90, 5); Produto p2 = new Produto("Caneta", 3.50, 10); p1.exibir(); p2.exibir(); } }
Saída esperada no console
=== Produto === Nome: Caderno Preço: R$ 12.9 Quantidade: 5 === Produto === Nome: Caneta Preço: R$ 3.5 Quantidade: 10
8)
Construtor Padrão
Básico
Crie a classe Animal com um construtor sem parâmetros que define valores padrão e um método apresentar().
Requisitos
  • Declare String nome e String especie
  • Construtor padrão: this.nome = "Sem nome" e this.especie = "Desconhecida"
  • Método void apresentar() imprime nome e espécie no formato Animal: nome (especie)
  • Instancie e chame apresentar()
Solução
public class Animal { String nome; String especie; public Animal() { this.nome = "Sem nome"; this.especie = "Desconhecida"; } void apresentar() { System.out.println("Animal: " + nome + " (" + especie + ")"); } } public class Ex08ConstrutorPadrao { public static void main(String[] args) { Animal a = new Animal(); a.apresentar(); } }
Saída esperada no console
Animal: Sem nome (Desconhecida)
9)
Sobrecarga de Construtores
Básico–Médio
Adicione à classe Animal um segundo construtor que recebe nome e especie como parâmetros. Crie dois objetos — um com cada construtor — e chame apresentar() em ambos.
Requisitos
  • Mantenha o construtor padrão da questão anterior
  • Adicione public Animal(String nome, String especie) que atribui os parâmetros
  • Crie a1 = new Animal() e a2 = new Animal("Rex", "Cachorro")
  • Chame apresentar() nos dois objetos
Solução
public class Animal { String nome; String especie; // Construtor 1 — sem parâmetros public Animal() { this.nome = "Sem nome"; this.especie = "Desconhecida"; } // Construtor 2 — com parâmetros (sobrecarga) public Animal(String nome, String especie) { this.nome = nome; this.especie = especie; } void apresentar() { System.out.println("Animal: " + nome + " (" + especie + ")"); } } public class Ex09Sobrecarga { public static void main(String[] args) { Animal a1 = new Animal(); // construtor 1 Animal a2 = new Animal("Rex", "Cachorro"); // construtor 2 a1.apresentar(); a2.apresentar(); } }
Saída esperada no console
Animal: Sem nome (Desconhecida) Animal: Rex (Cachorro)
Sobrecarga (overloading): Mesmos nome, parâmetros diferentes. O Java escolhe qual construtor invocar com base nos argumentos passados em new Animal(...).
10)
Classe Completa com POO
Básico–Médio
Crie a classe Pessoa aplicando todos os conceitos: atributos privados, construtor, getters/setters com validação e um método apresentar(). Demonstre o uso completo no main.
Requisitos
  • Atributos privados: private String nome e private int idade
  • Construtor public Pessoa(String nome, int idade) com this
  • Getters: getNome() e getIdade()
  • Setters: setNome() e setIdade() — o de idade valida se idade >= 0
  • Método void apresentar() imprime nome e idade
  • No main: crie um objeto, apresente, altere nome e idade, apresente novamente e tente setIdade(-5)
Solução
public class Pessoa { private String nome; private int idade; public Pessoa(String nome, int idade) { this.nome = nome; this.idade = idade; } public String getNome() { return nome; } public int getIdade() { return idade; } public void setNome(String nome) { this.nome = nome; } public void setIdade(int idade) { if (idade >= 0) this.idade = idade; else System.out.println("Idade inválida!"); } public void apresentar() { System.out.println("Nome: " + nome + " | Idade: " + idade); } } public class Ex10ClasseCompleta { public static void main(String[] args) { Pessoa p = new Pessoa("Maria", 25); p.apresentar(); p.setNome("Maria Silva"); p.setIdade(26); p.apresentar(); p.setIdade(-5); // deve exibir mensagem de erro } }
Saída esperada no console
Nome: Maria | Idade: 25 Nome: Maria Silva | Idade: 26 Idade inválida!
Conceitos aplicados: Encapsulamento (private), construtor com this, getters para leitura segura, setters com validação de negócio e método de comportamento. Esse é o padrão fundamental da POO em Java.
