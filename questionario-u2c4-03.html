1)
CriarClasseAbstrata
Básico
Crie a classe abstrata Funcionario com atributo nome, método abstrato calcularSalario() e método concreto exibirSalario().
Requisitos
  • Declare public abstract class Funcionario
  • Atributo private String nome, construtor e getNome()
  • Método abstrato: public abstract double calcularSalario()
  • Método concreto: exibirSalario() que imprime nome + ": R$ " + calcularSalario()
Solução
public abstract class Funcionario { private String nome; public Funcionario(String nome) { this.nome = nome; } public String getNome() { return nome; } // método abstrato — subclasses DEVEM implementar public abstract double calcularSalario(); // método concreto — herdado por todas as subclasses public void exibirSalario() { System.out.println(nome + ": R$ " + calcularSalario()); } }
2)
ImplementarGerente
Básico
Crie a classe Gerente que estende Funcionario e calcula o salário como salarioBase × 1.5 (bônus de 50%).
Requisitos
  • Atributo private double salarioBase
  • Construtor recebe nome e salarioBase, chama super(nome)
  • Implemente calcularSalario() com @Override retornando salarioBase * 1.5
Solução
public class Gerente extends Funcionario { private double salarioBase; public Gerente(String nome, double salarioBase) { super(nome); this.salarioBase = salarioBase; } @Override public double calcularSalario() { return salarioBase * 1.5; // 50% de bônus } }
Testando
Gerente g = new Gerente("Carlos", 5000.0); g.exibirSalario(); // herda Funcionario.exibirSalario() — polimorfismo! // Carlos: R$ 7500.0
3)
ImplementarVendedor
Básico
Crie a classe Vendedor que estende Funcionario e calcula o salário como salarioBase + comissao.
Requisitos
  • Atributos private double salarioBase e private double comissao
  • Construtor recebe nome, salarioBase e comissao, chama super(nome)
  • Implemente calcularSalario() retornando salarioBase + comissao
Solução
public class Vendedor extends Funcionario { private double salarioBase; private double comissao; public Vendedor(String nome, double salarioBase, double comissao) { super(nome); this.salarioBase = salarioBase; this.comissao = comissao; } @Override public double calcularSalario() { return salarioBase + comissao; } }
Testando
Vendedor v = new Vendedor("Maria", 2000.0, 1500.0); v.exibirSalario(); // Maria: R$ 3500.0
4)
AssociacaoSimples
Básico
Implemente uma Associação: a classe Medico usa a classe Paciente para realizar consultas, sem possuí-la.
Requisitos
  • Crie Paciente com atributo nome e getNome()
  • Em Medico, adicione consultar(Paciente p) que recebe Paciente como parâmetro
  • Crie Médico e Paciente de forma independente e demonstre que ambos existem sem dependência
Paciente.java
public class Paciente { private String nome; public Paciente(String nome) { this.nome = nome; } public String getNome() { return nome; } }
Medico.java
public class Medico { private String nome; public Medico(String nome) { this.nome = nome; } // Associação: usa Paciente via parâmetro — não cria, não possui public void consultar(Paciente p) { System.out.println("Dr. " + nome + " atendeu " + p.getNome()); } }
Testando — ciclos de vida independentes
Medico m = new Medico("Dra. Ana"); // criado independentemente Paciente p = new Paciente("João"); // criado independentemente m.consultar(p); // Dra. Ana atendeu João // Ambos continuam existindo após a consulta!
5)
AgregacaoTurmaAluno
Básico
Implemente uma Agregação: a classe Turma agrega Alunos que existem independentemente dela.
Requisitos
  • Crie Aluno com nome e getNome()
  • Em Turma, use List<Aluno> e método addAluno(Aluno a) — Aluno criado fora
  • Crie os Alunos antes da Turma e adicione via addAluno(), demonstrando ciclo de vida independente
Aluno.java
public class Aluno { private String nome; public Aluno(String nome) { this.nome = nome; } public String getNome() { return nome; } }
Turma.java
public class Turma { private String nome; private List<Aluno> alunos = new ArrayList<>(); public Turma(String nome) { this.nome = nome; } // Agregação: Aluno criado FORA e passado como parâmetro public void addAluno(Aluno a) { alunos.add(a); } public void listar() { System.out.println("Turma: " + nome); for (Aluno a : alunos) System.out.println(" - " + a.getNome()); } }
Testando — Aluno existe antes da Turma!
Aluno a1 = new Aluno("Carlos"); // criado independentemente Aluno a2 = new Aluno("Maria"); Turma t = new Turma("Java 101"); t.addAluno(a1); t.addAluno(a2); t.listar(); // Turma: Java 101 // - Carlos // - Maria // Se t = null → a1 e a2 continuam existindo!
6)
ComposicaoPedidoItem
Intermediário
Implemente uma Composição: a classe Pedido cria e controla seus ItemPedidos internamente — eles não existem fora do pedido.
Requisitos
  • Crie ItemPedido com produto (String) e preco (double)
  • Em Pedido, método addItem(String produto, double preco) cria ItemPedido internamente
  • Adicione calcularTotal() que soma todos os preços dos itens
ItemPedido.java
public class ItemPedido { private String produto; private double preco; public ItemPedido(String produto, double preco) { this.produto = produto; this.preco = preco; } public double getPreco() { return preco; } }
Pedido.java
public class Pedido { private int numero; private List<ItemPedido> itens = new ArrayList<>(); public Pedido(int numero) { this.numero = numero; } // Composição: ItemPedido criado DENTRO do Pedido! public void addItem(String produto, double preco) { itens.add(new ItemPedido(produto, preco)); } public double calcularTotal() { double total = 0; for (ItemPedido i : itens) total += i.getPreco(); return total; } }
Testando — ItemPedido existe apenas dentro do Pedido
Pedido p = new Pedido(1); p.addItem("Teclado", 150.0); p.addItem("Mouse", 80.0); System.out.println("Total: R$ " + p.calcularTotal()); // Total: R$ 230.0 // Se p = null → os ItemPedidos somem junto!
7)
PolimorfismoFuncionarios
Intermediário
Use polimorfismo com Funcionario[] para calcular a folha de pagamento total de uma equipe com Gerentes e Vendedores.
Requisitos
  • Crie um array Funcionario[] com pelo menos 4 funcionários (misture Gerentes e Vendedores)
  • Use for (Funcionario f : equipe) e chame f.exibirSalario() em cada um
  • Acumule o total somando f.calcularSalario() e imprima a folha
Solução
Funcionario[] equipe = { new Gerente("Alice", 6000.0), new Vendedor("Bob", 2000.0, 1500.0), new Gerente("Carol", 5000.0), new Vendedor("Dan", 2000.0, 800.0) }; double folha = 0; for (Funcionario f : equipe) { f.exibirSalario(); // polimorfismo: cada um usa SUA versão! folha += f.calcularSalario(); } System.out.printf("Folha total: R$ %.2f%n", folha);
Saída esperada
Alice: R$ 9000.0 Bob: R$ 3500.0 Carol: R$ 7500.0 Dan: R$ 2800.0 Folha total: R$ 22800,00
8)
ComposicaoEmpresaDepartamento
Intermediário
Modele uma Empresa composta por Departamentos. Os departamentos não existem fora da empresa — composição forte.
Requisitos
  • Crie Departamento com nome e responsavel (String)
  • Em Empresa, crie cada Departamento internamente via addDepartamento(String nome, String resp)
  • Adicione listar() que exibe o nome da empresa e todos os departamentos
Departamento.java
public class Departamento { private String nome; private String responsavel; public Departamento(String nome, String responsavel) { this.nome = nome; this.responsavel = responsavel; } public String getNome() { return nome; } public String getResponsavel() { return responsavel; } }
Empresa.java
public class Empresa { private String nome; private List<Departamento> departamentos = new ArrayList<>(); public Empresa(String nome) { this.nome = nome; } // Composição: Departamento criado DENTRO da Empresa! public void addDepartamento(String nome, String resp) { departamentos.add(new Departamento(nome, resp)); } public void listar() { System.out.println("Empresa: " + nome); for (Departamento d : departamentos) System.out.println(" [" + d.getNome() + "] → " + d.getResponsavel()); } }
Saída
Empresa e = new Empresa("TechCo"); e.addDepartamento("TI", "Alice"); e.addDepartamento("RH", "Bob"); e.listar(); // Empresa: TechCo // [TI] → Alice // [RH] → Bob
9)
DiferencaAgregacaoComposicao
Intermediário
Demonstre na prática a diferença entre Agregação e Composição: um Aluno (agregação) sobrevive à destruição de uma Turma, mas um ItemPedido (composição) não existiria sem o Pedido.
Requisitos
  • Crie um Aluno e adicione-o a uma Turma
  • Simule a destruição da turma com t = null
  • Mostre que o Aluno ainda existe e pode ser acessado (agregação)
  • Comente como seria a composição (ItemPedido some com o Pedido)
Demonstrando Agregação vs Composição
// ─── AGREGAÇÃO: Aluno tem vida independente ──────────────── Aluno a1 = new Aluno("Carlos"); // criado FORA da turma Turma t = new Turma("Java 101"); t.addAluno(a1); System.out.println("Aluno na turma: " + a1.getNome()); // Carlos t = null; // "destruindo" a turma... System.out.println("Aluno após t=null: " + a1.getNome()); // Carlos ✓ // ─── COMPOSIÇÃO: ItemPedido não sobreviveria ─────────────── // Pedido p = new Pedido(1); // p.addItem("Teclado", 150.0); // criado DENTRO do pedido // p = null; → ItemPedido some junto! (sem referência externa)
Saída
Aluno na turma: Carlos Aluno após t=null: Carlos ✓ → Agregação: partes sobrevivem ao todo. → Composição: partes são destruídas com o todo.
10)
SistemaCompletoAbstrataAssociacaoComposicao
Intermediário
Combine os três conceitos: Funcionario (abstrata + polimorfismo), Projeto (Associação com Funcionario) e Empresa (Composição de Departamentos).
Requisitos
  • Crie Projeto com nome e método atribuir(Funcionario f) que exibe a atribuição (Associação)
  • Crie pelo menos um Gerente e um Vendedor e os atribua ao projeto
  • Use um array Funcionario[] para exibir a folha via polimorfismo
Projeto.java
public class Projeto { private String nome; public Projeto(String nome) { this.nome = nome; } // Associação: usa Funcionario via parâmetro — não o possui public void atribuir(Funcionario f) { System.out.println(f.getNome() + " → projeto: " + nome); } }
Combinando tudo no main
// Hierarquia abstrata Gerente g = new Gerente("Alice", 6000.0); Vendedor v = new Vendedor("Bob", 2000.0, 1500.0); // Associação: Projeto usa Funcionários sem possuí-los Projeto p1 = new Projeto("Sistema de Vendas"); p1.atribuir(g); p1.atribuir(v); // Polimorfismo com a hierarquia abstrata Funcionario[] equipe = { g, v }; for (Funcionario f : equipe) { f.exibirSalario(); // despacho dinâmico! }
Saída — três conceitos funcionando juntos
Alice → projeto: Sistema de Vendas Bob → projeto: Sistema de Vendas Alice: R$ 9000.0 Bob: R$ 3500.0
