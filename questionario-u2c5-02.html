📋 Quiz 02 — throw, throws e Exceções Personalizadas
Aula 05 — Tratamento de Exceções  ·  10 questões de Verdadeiro ou Falso
🛠 throw / throws 🟣 Médio ⏱ ~5 min
0 / 10 respondidas
01
A palavra-chave throw é usada para lançar manualmente uma exceção, geralmente com throw new Exception(...).
02
A palavra-chave throws, usada no corpo do método, serve para capturar uma exceção que foi lançada.
03
Um método que declara throws Exception obriga quem o chama a tratar essa exceção com try-catch ou propagá-la com throws também.
04
Para criar uma exceção personalizada, a classe deve estender (extends) a classe Exception (ou uma de suas subclasses).
05
No construtor de uma exceção personalizada, super(mensagem) repassa a mensagem de erro ao construtor da superclasse Exception.
06
O método e.getMessage() retorna o stack trace completo da exceção, e não apenas a mensagem de erro.
07
IOException e SQLException são exemplos de exceções checked — devem ser tratadas ou declaradas obrigatoriamente.
08
NullPointerException, ArithmeticException e ArrayIndexOutOfBoundsException são exemplos de exceções unchecked (subclasses de RuntimeException).
09
É obrigatório capturar (catch) uma exceção unchecked para que o código compile.
10
Uma exceção personalizada, como MinhaExcecao, pode ser usada em um bloco catch específico: catch (MinhaExcecao e) { ... }.
