1)
TratarDivisaoPorZero
Básico
Escreva um trecho que tenta dividir dois números inteiros e trata o erro de divisão por zero com try-catch.
Requisitos
  • Declare int a = 10, b = 0;
  • Dentro do try, calcule a / b
  • Capture ArithmeticException e imprima "Erro: divisão por zero."
Solução
try { int a = 10, b = 0; int resultado = a / b; System.out.println("Resultado: " + resultado); } catch (ArithmeticException e) { System.out.println("Erro: divisão por zero."); }
Saída
Erro: divisão por zero.
2)
TratarAcessoArrayInvalido
Básico
Escreva um trecho que acessa uma posição inválida de um array e trata o erro com try-catch.
Requisitos
  • Declare int[] numeros = {1, 2, 3};
  • Dentro do try, acesse numeros[5] (índice inexistente)
  • Capture ArrayIndexOutOfBoundsException e imprima "Erro: índice inválido!"
Solução
try { int[] numeros = {1, 2, 3}; System.out.println(numeros[5]); } catch (ArrayIndexOutOfBoundsException e) { System.out.println("Erro: índice inválido!"); }
Saída
Erro: índice inválido!
3)
TratarObjetoNulo
Básico
Escreva um trecho que tenta chamar um método em uma referência null e trata o erro com try-catch.
Requisitos
  • Declare String texto = null;
  • Dentro do try, chame texto.length()
  • Capture NullPointerException e imprima "Erro: objeto não inicializado."
Solução
try { String texto = null; System.out.println(texto.length()); } catch (NullPointerException e) { System.out.println("Erro: objeto não inicializado."); }
Saída
Erro: objeto não inicializado.
4)
TratarLeituraArquivo
Básico
Escreva um trecho que tenta abrir um arquivo inexistente e trata a exceção de leitura com try-catch.
Requisitos
  • Dentro do try, crie new FileReader("dados.txt")
  • Capture IOException
  • Imprima "Erro ao ler o arquivo: " + e.getMessage()
Solução
try { FileReader arquivo = new FileReader("dados.txt"); } catch (IOException e) { System.out.println("Erro ao ler o arquivo: " + e.getMessage()); }
Saída (se "dados.txt" não existir)
Erro ao ler o arquivo: dados.txt (Arquivo não encontrado)
5)
MultiplosCatch
Intermediário
Crie o método dividirElementos que acessa um elemento do array e o divide por um valor, tratando dois tipos de exceção em blocos catch separados.
Requisitos
  • static void dividirElementos(int[] arr, int indice, int divisor)
  • Dentro do try, calcule arr[indice] / divisor e imprima o resultado
  • Adicione um catch para ArrayIndexOutOfBoundsException e outro para ArithmeticException
Solução
static void dividirElementos(int[] arr, int indice, int divisor) { try { int resultado = arr[indice] / divisor; System.out.println("Resultado: " + resultado); } catch (ArrayIndexOutOfBoundsException e) { System.out.println("Erro: índice inválido!"); } catch (ArithmeticException e) { System.out.println("Erro: divisão por zero!"); } }
Testando
int[] valores = {10, 20, 30}; dividirElementos(valores, 5, 2); // Erro: índice inválido! dividirElementos(valores, 0, 0); // Erro: divisão por zero! dividirElementos(valores, 1, 4); // Resultado: 5
6)
BlocoFinally
Intermediário
Demonstre o uso do bloco finally, que executa sempre — com ou sem exceção — simulando o fechamento de um recurso.
Requisitos
  • No try, simule abrir um recurso (imprima "Recurso aberto.") e provoque uma ArithmeticException
  • No catch, trate a exceção
  • No finally, imprima "Recurso fechado." — deve executar sempre
Solução
try { System.out.println("Recurso aberto."); int resultado = 10 / 0; // provoca exceção } catch (ArithmeticException e) { System.out.println("Erro: divisão por zero."); } finally { System.out.println("Recurso fechado."); // SEMPRE executa }
Saída
Recurso aberto. Erro: divisão por zero. Recurso fechado.
7)
LancarExcecaoComThrow
Intermediário
Crie o método verificarIdade que lança uma exceção caso a idade seja menor que 18, usando throw e throws.
Requisitos
  • public static void verificarIdade(int idade) throws Exception
  • Se idade < 18, lance throw new Exception("Idade inválida! Deve ser maior que 18.")
  • No chamador, use try-catch para tratar a exceção lançada
verificarIdade
public static void verificarIdade(int idade) throws Exception { if (idade < 18) { throw new Exception("Idade inválida! Deve ser maior que 18."); } }
Tratando a exceção lançada
try { verificarIdade(15); } catch (Exception e) { System.out.println("Erro: " + e.getMessage()); }
Saída
Erro: Idade inválida! Deve ser maior que 18.
8)
CriarExcecaoPersonalizada
Intermediário
Crie a classe MinhaExcecao, uma exceção personalizada que estende Exception.
Requisitos
  • public class MinhaExcecao extends Exception
  • Construtor que recebe String mensagem
  • No construtor, chame super(mensagem) para repassar a mensagem à superclasse
MinhaExcecao.java
public class MinhaExcecao extends Exception { public MinhaExcecao(String mensagem) { super(mensagem); // repassa a mensagem ao construtor de Exception } }
Por que extends Exception?
Herdar de Exception faz MinhaExcecao se comportar como qualquer exceção padrão do Java: pode ser lançada com "throw", declarada com "throws" e capturada com "catch (MinhaExcecao e)".
9)
UsarExcecaoPersonalizada
Intermediário
Usando a classe MinhaExcecao do exercício anterior, crie o método validarNota que valida se uma nota está entre 0 e 10.
Requisitos
  • static void validarNota(int nota) throws MinhaExcecao
  • Se nota < 0 || nota > 10, lance throw new MinhaExcecao("Nota inválida! Deve estar entre 0 e 10.")
  • No main, chame validarNota(15) dentro de um try-catch (MinhaExcecao e)
Solução
static void validarNota(int nota) throws MinhaExcecao { if (nota < 0 || nota > 10) { throw new MinhaExcecao("Nota inválida! Deve estar entre 0 e 10."); } } public static void main(String[] args) { try { validarNota(15); } catch (MinhaExcecao e) { System.out.println("Erro: " + e.getMessage()); } }
Saída
Erro: Nota inválida! Deve estar entre 0 e 10.
10)
SistemaCompletoValidacaoCadastro
Intermediário
Combine os conceitos: crie a exceção personalizada DadosInvalidosException e um método cadastrarUsuario que valida nome e idade, lançando a exceção quando os dados forem inválidos.
Requisitos
  • Crie DadosInvalidosException extends Exception com construtor super(mensagem)
  • static void cadastrarUsuario(String nome, int idade) throws DadosInvalidosException
  • Se nome estiver vazio, lance a exceção com mensagem específica; se idade < 18, lance com outra mensagem
  • No main, trate com try-catch e imprima o erro
DadosInvalidosException.java
public class DadosInvalidosException extends Exception { public DadosInvalidosException(String mensagem) { super(mensagem); } }
cadastrarUsuario — múltiplas validações
static void cadastrarUsuario(String nome, int idade) throws DadosInvalidosException { if (nome == null || nome.isEmpty()) { throw new DadosInvalidosException("Nome não pode ser vazio."); } if (idade < 18) { throw new DadosInvalidosException("Idade mínima é 18 anos."); } System.out.println("Usuário " + nome + " cadastrado com sucesso!"); }
Testando no main
try { cadastrarUsuario("Ana", 15); } catch (DadosInvalidosException e) { System.out.println("Erro no cadastro: " + e.getMessage()); }
Saída
Erro no cadastro: Idade mínima é 18 anos.
